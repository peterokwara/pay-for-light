    MAM API v1.0.0

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    POST /v0/publish

    REST call to perform a MAM.publish

    Body parameter

    {
  "provider": "devnet",
  "depth": 3,
  "mwm": 9,
  "mode": "restricted",
  "key": "ABCDEFG9ABCDEFG",
  "seed": "MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAMAPI9MAAA",
  "index": 3,
  "dataType": "json",
  "data": {
    "a": "hello",
    "b": 2
  },
  "tag": "MAM9MAM9MAM"
}

    Parameters

    Name In Type Required Description
    provider body string true This can either be either of the constants mainnet, devnet or the url of a node e.g. https://nodes.thetangle.org:443
    depth body number false This depth at which to attach the MAM transaction to the tangle, this is only used when the provider is a url
    mwm body number false The mwm with which to attach the MAM transaction to the tangle, this is only used when the provider is a url
    mode body string true The mode for the MAM channel either public, private or restricted
    key body string false This key is required when the mode is restricted, it should be trytes
    seed body string true This seed which is used to generate the channel, it should be trytes and 81 characters in length
    index body number false The index to publish the message in the channel, defaults to 0 if not provided
    dataType body string true The type of the data in the data field either trytes, text or json
    data body any true The type of the data in the data field either trytes, text or json
    » anonymous body string false none
    » anonymous body object false none
    tag body string false This tag which is set on the MAM message, it should be trytes <= 27 characters in length

    Enumerated Values

    Parameter Value
    mode public
    mode private
    mode restricted
    dataType trytes
    dataType text
    dataType json

    GET /v0/fetch

    REST call to perform a MAM.fetchSingle

    Parameters

    Name In Type Required Description
    provider query string true This can either be either of the constants mainnet, devnet or the url of a node e.g. https://nodes.thetangle.org:443
    mode query string true The mode for the MAM channel either public, private or restricted
    key query string true This key is required when the mode is restricted, it should be trytes
    root query string true The root is where we should read the MAM message from
    dataType query string true The type of the data in the data field either trytes, text or json

    Example responses

    200 Response

    {
  "success": true,
  "message": "OK",
  "data": "HELLO",
  "nextRoot": "MMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMMM",
  "tag": "FOO"
}

    Responses

    Status Meaning Description Schema
    200 OK Success Inline
    400 Bad Request Failure Inline
    404 Not Found There was no data found on the requested root Inline

    Response Schema

    Status Code 200

    Name Type Required Restrictions Description
    » success boolean true none True if the operation was a success, false if it failed
    » message string true none A message explaining the result of the operation
    » data string true none The data that was found at the requested root
    » nextRoot string true none The next root in the channel
    » tag string false none The tag stored with the message

    Status Code 400

    Name Type Required Restrictions Description
    » success boolean true none True if the operation was a success, false if it failed

    Name Type Required Restrictions Description

    » message string true none A message explaining the result of the operation
    This operation does not require authentication

    Example

    To publish and fetch a message from a MAM channel using the REST API.

    const fetch = require('node-fetch');

const API_ENDPOINT = 'https://mam-api.dag.sh';

const SEED = 'YOUR_SEED_HERE' // 81 Trytes;

const provider = "devnet";
const mode = "restricted";
const key = "ABCDEFG9ABCDEFG";

fetch(
    `${API_ENDPOINT}/v0/publish`,
    {
        method: "POST",
        headers: {
            'Content-Type': 'application/json',
            'Accept': 'application/json'
        },
        body: JSON.stringify({
            provider,
            mode,
            key,
            seed: SEED,
            dataType: "json",
            data: {
                "a": "hello",
                "b": 2
            },
            tag: "MYMAM"
        })
    })
    .then((res) => {
        return res.json();
    }).then((publishResponse) => {
        if (publishResponse.success) {
            console.log("Publish Success");
            console.log("Published root", publishResponse.publishedRoot);
            console.log("Next index to publish", publishResponse.nextIndex);
            console.log("View on Tangle", `https://utils.iota.org/mam/${publishResponse.publishedRoot}/${mode}/${key}/${provider}`);

            fetch(
                `${API_ENDPOINT}/v0/fetch?provider=${provider}&mode=${mode}&key=${key}&root=${publishResponse.publishedRoot}&dataType=json`,
                {
                    method: "GET",
                    headers: {
                        'Accept': 'application/json'
                    }
                }).then((res) => {
                    return res.json();
                }).then((fetchResponse) => {
                    if (fetchResponse.success) {
                        console.log("Fetch Success");
                        console.log("Data", fetchResponse.data);
                    }
                })
                .catch((err) => console.error("Error during request", err))
        }
    })
    .catch((err) => console.error("Error during request", err));
